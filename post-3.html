Arrays & Hashes

3/4/16

Hi again, folks! This week we dove into the wonderful world of programming in Ruby. There are a handful of different classes of data in Ruby - For instance, integers are whole numbers: (20,-3) and strings are string of letters ("Hello world"). Today, we're going to be talking about arrays and hashes, and why you'd use one over the other.

Arrays

Arrays are ordered collections of objects in Ruby. You can have an array of strings, integers, floats, booleans- pretty much anything you like. You can use different types of objects in the same array too. There are a few different ways to create an array. Let's take a look at the syntax:

#Creates an array filled with strings

favorite_foods = ["pizza", "chocolate", "pasta", "tacos"]

#Your array can have different data types: variables, strings, integers, booleans, etc.

favorite_foods_and_numbers = [4, "pizza", "chocolate", 29.2, true]

#Creates a new empty array favorite_foods = Array.new

#Creates an empty array with a size of 4. favorite_foods = Array.new(4)

#Creates a new array filled with strings

favorite_foods = Array.new("pizza", "chocolate", "pasta", "tacos")

Hashes

Hashes, on the other hand, are a collection of key-value pairs. They act sort of like a dictionary- associating a category with a value. You would use a hash rather than an array in the event that you'd want to store pairs of information rather than just one set of information. Check out the Ruby hash syntax:

#Creates a hash filled with strings

favorite_foods = {
"Christyn" => "pizza"
"Dottie" => "dog food"
"Charlie" => "chocolate"
}

#Your hash can have different data types

favorite_numbers = {
"Christyn" => 8
"Dottie" => 1
"Charlie" => 3
}

#An alternate syntax is to use keys that are symbols

favorite_foods = {
:Christyn => "pizza"
:Dottie" => "dog food"
:Charlie" => "chocolate"
}

#Even another way to write it

favorite_foods = {
Christyn: "pizza"
Dottie: "dog food"
Charlie: "chocolate"
}

#You can also create hashes using Hash.new

favorite_foods = Hash.new
