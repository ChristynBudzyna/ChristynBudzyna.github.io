Enumerable Methods in Ruby

3/12/16

This week we did a deeper dive into Ruby and the methods certain objects are able to perform. There are a few different kinds of collection objects in Ruby, including arrays, hashes, ranges, and sets. Collection objects in Ruby typically include what is called the Enumerable module. The Enumerable module supplies objects of collection classes with different collection related behaviors. You can also create your own classes and equip them with the Enumerable methods. The methods behind these behaviors function in terms of "each" element in the collection. Today I'm going to talk about two of the most common Enumerable methods, and the difference between them.

.Each Iterator

Enumerable methods perform different functions for different sets of collections. I'll use two of the more common collections, arrays and hashes, as examples. If you use .each with an array, it iterates over the array and runs the block for each value. It returns the original object it was called on.


#Yields modified array [6,7,8,9,10]

If you use .each with an hash, it iterates over the keys and values of a hash. In this case, it modifies the values of the hash.


#Yields strings "In two years, Dottie will be 4", "In two years Coco will be 17."

.Map Iterator

.Map differs from .each in that it returns an entirely new object (an array), instead of just modifying the initial object. It runs the block once for each element of the collection, and then outputs a brand new array.


#Yields brand new array [6,7,8,9,10]. The function may look similar to .each, but it's outputting a new object.

If you use .map with a hash, it will output a brand new array from the hash


Outputs a new array of [3,17]
