Intro to Ruby on Rails

4/2/16

What is Rails?

Ruby on Rails is a framework for creating web applications, written in the Ruby language. It's designed to make it easier for programmers to create apps, as it makes a series of assumptions about how most programmers will want to design an app. The series of conventions are called the Rails API (application programming interface). When you're learning Rails, you're really learning how to use the Rails conventions and its API. Technically, Rails is a package library (a Ruby Gem) that is installed using the command line.

Rails utilizes four different languages to create a web application: HTML, CSS, and Javascript as front end languages, and Ruby as a back end language.

Rails Philosophy

The Rails philosophy has two basic principles:

  1. DRY(Don't repeat Yourself): This keeps your code more maintainable and less prone to bugs
  2. Convention over Configuration: There are a lot of default settings, so you don't have to specify every detail of your program.

    3. Rails Design Concept

    Rails is built around the MVC or model-view controller pattern. It's a way to separate the logic, data, and display in your program. This is the basic logic to how Rails works using the MVC:

    1. A browser makes a request, e.g. the request of navigating to a new page
    2. A web server receives the request, and decides which controller to use.
    3. The web server uses the dispatcher to create a new controller.
    4. The controller analyzes user requests and data submissions, and asks the model to fetch the requested information.
    5. The models store data in the database, perform logic, and retrieve information from the database.
    6. The views read what the controller gives to them in the form of HTML and CSS.
    7. The controller returns data to the server, which sends it to the user
