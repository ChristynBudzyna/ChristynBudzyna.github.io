The Command Line

If you open the terminal on your computer, (command-spacebar, type in "terminal"), a magical box of wonders appears! Well, maybe it doesn't look so magical at first, but it can do amazing things. You can think of the command line as a super charged version of your "Finder." In your terminal, you can navigate to all the files in your computer, find out information about them, create, delete, copy, and edit files and directories (a fancy word for folder), search for text in a file, change permissions on a file, run code, and SO MUCH MORE! We focused on repeating and memorizing the basic navigational commands. Here are a few of them you can try out.

  • pwd - Print working directory - This shows you the directory you are currently in.
  • hostname - Show my computer's network name
  • mkdir folder_name - Make a new directory
  • rmdir folder_name - Remove a directory
  • touch file_name - Create a file
  • rm file_name - Remove a file
  • ls - List everything inside a directory
  • cd - Change to a new directory
  • cp - Copy a file or directory
  • Mv - Move a file or directory

I have a list of about 30-40 commands that I am working to commit to memory. Using the command line really makes you feel powerful and in control of your computer.

Version Control- Git and GitHub

The other big theme of the week was version control. Programmers use version control to keep track of all of the different drafts of a file. This prevents confusion when they want to go back to a previous version of a file. This also allows multiple people to be working on a file without getting in each other's way. Using the terminal, you can install software called Git to any directory on your computer, and it will track each version of the file. You need to remember to create saving points or "commits" when you make a change you'd like to save. Thereâ€™s a four step process for this.

  1. The first step is to initialize git on your directory. You can either clone your directory from github using "git clone file_name" or initialize it on your machine using "git init file_name"
  2. Once you make your changes to the file, check the status using "git status."" It should say that there is a file that needs editing.
  3. Use "git add file_name" to add your file to the staging area. You can add all your files using "git add ."
  4. Use "git commit -m "Message here" to commit your changes and save your work!

It's also a good idea to store your code on the website, Github. This creates a version of your file that lives in the cloud, so you'll still have your work in case your computer crashes and burns in some horrible accident. You can also share your code with the world on Github!

Use GitHub and don't lose your code if this happens!
