Positioning in CSS

2/26/16

Hi folks! The past couple of weeks at Dev Bootcamp have been all about HTML and CSS. I have some experience using both, but I was able to learn and explore some new concepts. For those of you who are new to coding, HTML is the language you use to build the blocks of a website, and CSS is the language you use to style it. Last week, I built the bones of this website in HTML, and this past Monday, I styled it in CSS. I find CSS a little more fun to write than HTML, because it exercises the creative side of my brain a little more. Most of CSS is fairly straightforward to me, but positioning elements on a page is the concept that I have found to be the trickiest. I'm going to break down positioning in CSS for you, so we can both get a better grasp on it!

Listen to Bob Ross.

Position Property

The position property specifies the type of positioning method an element will use. There are four different types of positions that an element can have: static, relative, absolute, and fixed. Let's say we have an image tag that holds a picture of my dog, Dottie. I am assigning this image tag with an id of "dog" so I can select it in css with #dog.

<img src="imgs/dottie.png" id="dog">

Static Positioning

Static positioning is the default position in HTML. Because there are no other elements on this page, Dottie stays in the top left hand corner of the screen.

What this looks like in CSS:

#dog {
position: static;
}

Dottie ain't goin' nowhere in static positioning.

Relative Positioning

In relative positioning, the image moves relative of where it would have been in static positioning. Note: this is NOT relative to the other elements on the screen, it is only relative to where it would be.

What this looks like in CSS:

#dog {
position: relative;
top: 100px;
}

This moves Dottie 100 pixels down from the top of where she would have been in static positioning.

Absolute Positioning

In absolute positioning, there are two possible options. If the element is the only thing on the page with absolute positioning, then it is positioned in terms of the entire page. If there are more than one elements without a static position, the element is positioned in relation to the first parent element that doesn't have position:static.

What this looks like in CSS:

#dog {
position:absolute;
top:100px;
right:100px;
}

Dottie is positioned 100 pixels from the top of the whole page, and 100 pixels right of the whole page. If there were other elements not in the static position, she would be positioned in relation to them.

Fixed Positioning

An element in fixed positioning is positioned relative to the viewport (the user's visable area of a web page). It will always stay in the same place even if the page is scrolled.

What this looks like in CSS:

#dog {
position:fixed;
}

Dottie stays in the same spot, even though I've scrolled down.
